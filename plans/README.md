## Findings considered and rejected

- **`normalizeWithMap` relies on `.trim()` being a no-op** (`src/index.ts:134`):
  the normalized text never has leading/trailing spaces by construction, so
  `.trim()` removes nothing and the `text`/`map` arrays stay aligned. Fragile to
  read but currently correct; not worth a change without a reproducing failure.
- **CLI reads the whole file synchronously with no stdin path** (`src/cli.ts:13`):
  fine for a small local utility; streaming/stdin is a nice-to-have, not a defect.
  Re-evaluate only if large-file CLI use becomes a real workflow.
- **Fuzzy `step` sampling can miss the optimal window** (`src/index.ts:147`):
  documented as "small fuzzy window search" — a deliberate speed/accuracy
  tradeoff, not a bug. Plan 005 caps the cost rather than densifying the search.
